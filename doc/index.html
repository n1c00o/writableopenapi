Writable OpenAPI

A Domain Specific Language to write OpenAPI specifications.

OpenAPI is a useful specification to define HTTP APIs. It is used by many tools to generate documentation, clients, servers, etc.
However, it is not very easy to write. Therefore, this project aims to provide a way to write OpenAPI specifications in a language that is more familiar to developers.

This is the reason why the project is using Python as it DSL, since Python is a very popular language and is very easy to write in.

Example

You can find a sample DSL here, with generated specifications in JSON and YAML.

Installation

After setting up a Python environment, you can install the package by running:


    pip install writableopenapi@git+https://github.com/n1c00o/writableopenapi

Then, simply describes your API using the various macros and classes.

Documentation

The OpenAPI specifications are defined within the writableopenapi.openapi package.

There are sets of macros – functions – to help writing the API documentation. They are available in the writableopenapi.macros package.

For instance, the writableopenapi.macros.logics package offers the possibility to easily write schemas that are based on logics, e.g. allOf, anyOf, oneOf...

The writableopenapi.macros.types package offers functions to define common type schemas, e.g. string, uuid...

The writableopenapi.macros.shared package lets define components that can be reused in the API documentation. The various classes can easily be used to get a reference to themselves.
